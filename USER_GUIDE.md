## 1. 도구 소개

NIKKE 유니온 레이드 진행 후, **블라블라링크 ShiftyPad**의 회차 데이터(레이드 결과 + 멤버 싱크로 레벨)를 **본인 Google Sheets 사본**에 자동 입력합니다.

자동 처리되는 내용:

- 시트에 빠진 과거 회차 자동 감지 + 다회차 백필 (블라블라링크 제공 범위 내)
- 레이드 결과 자동 입력 (보스명 / 단계 / 출진 니케 / 돌파 / 딜량 / 막타 여부)
- 회차 당시 싱크로 레벨 자동 입력 (각 회차 출전 니케 최고 레벨). 최신 회차 미참여자는 현재 싱크로 레벨로, 과거 미참여 회차는 빈 칸으로 둠
- 탈퇴 멤버 자동 제거 + 신규 가입자 자동 추가
- 백업 탭 자동 생성
- 탈퇴자 레벨 기록 (`탈퇴자 레벨 기록` 탭이 있는 시트 한정) — 유니온을 떠난 멤버의 회차별 레벨 보존

## 3. 최초 설정 (1회만)

### Step 1: Tampermonkey 설치

[Chrome 웹스토어](https://chromewebstore.google.com/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo)에서 Tampermonkey 확장프로그램을 설치합니다.

**중요**: Tampermonkey 설치 후 확장 페이지에서:

1. `chrome://extensions` 진입
2. 우측 상단 **"개발자 모드"** 켜기
3. **Tampermonkey 세부정보 → "사용자 스크립트 허용" 켜기**

이 설정 없으면 유저스크립트가 작동하지 않습니다.

### Step 2: 유저스크립트 설치

[Greasyfork 페이지](https://greasyfork.org/scripts/579278) 접속 → **[설치하기]** 버튼 → Tampermonkey 설치 확인 창에서 **[설치]** 클릭

### Step 3: NIKKE 유레 시트 본인 사본 생성

1. [arca.live 게시글](https://arca.live/b/nikketgv/161405505)에서 NIKKE 유레 시트 원본 링크 확인
2. 원본 시트 접속 → `파일 > 사본 만들기`
3. 본인 Google Drive에 저장

### Step 4: 유니온 멤버 입력 (최초 1회)

본인 사본의 **`유니온 멤버` 탭**에서 **가입 순서대로** 32명의 닉네임을 입력합니다.

```
A열 (가입 순서) | B열 (닉네임)
1              | 1번째 멤버 닉네임
2              | 2번째 멤버 닉네임
3              | ...
...            | ...
32             | 32번째 멤버 닉네임
```

> ⚠️ 이 단계는 **사용자가 직접** 입력해야 합니다. 이 도구는 가입 순서 정보를 알 수 없습니다.

### Step 5: 도구 페이지 접속 + 로그인

1. [https://ssissun.github.io/nikke-raid-autosync-spa/](https://ssissun.github.io/nikke-raid-autosync-spa/) 접속
2. **[Google 로그인]** 클릭 → 본인 Google 계정 선택
3. 권한 동의 화면에서 **"이 앱이 Google 계정에 액세스하도록 허용"** 클릭
4. 화면 상단에 `✅ 로그인됨 · [본인 이메일]` 표시 확인

### Step 6: 사본 시트 선택

1. **[시트 선택]** 클릭 → Google Picker 창이 열림
2. **본인이 생성한 사본 시트** 선택 → **[선택]** 클릭
3. 자동으로 시트 검증이 실행됨
4. `✅ 시트 구조 검증 통과` 표시 확인

**⚠️ "이 시트는 도구가 알고 있는 구조와 일치하지 않습니다" 알림이 뜬 경우** → [Step 7](#step-7-신뢰-다이얼로그-처리-필요-시) 참고

### Step 7: 신뢰 다이얼로그 처리 (필요 시)

본인 사본의 헤더가 도구가 알고 있는 템플릿과 일치하지 않으면 신뢰 다이얼로그가 표시됩니다. 다음 항목을 **직접 확인 후 모두 체크**:

- [ ] `유니온 멤버` 탭이 존재하며 헤더가 `가입 순서 / 닉네임` 구조이다
- [ ] `레이드 통계` 탭이 존재하며 16개 컬럼 헤더가 정확히 일치한다
- [ ] `레이드 결과` 탭이 존재하며 `회차` 컬럼이 있다
- [ ] 본인이 운영하는 NIKKE 유니온 레이드 시트이다
- [ ] 시트 헤더 컬럼명을 임의로 수정한 적 없다
- [ ] 손상 발생 시 본인이 복원 책임을 진다는 점에 동의한다

모두 체크 후 **[✅ 이 시트를 신뢰하고 진행]** 클릭 → 다음부터 자동 통과.

신뢰하지 않으면 상단 **[변경]** 버튼으로 다른 시트를 선택해주세요.

## 4. 회차마다 사용하기

### 사용 흐름 (2 클릭)

#### 1️⃣ 신규 회차 데이터 가져오기

도구 페이지에서 **[🎯 신규 회차 데이터 가져오기]** 클릭 → 블라블라링크 새 탭이 열림

→ 유저스크립트가 자동으로 데이터를 수집해서 도구 페이지로 전달 (시트에 빠진 과거 회차가 여러 개면 모두 함께 수집)

→ 도구 페이지에 **📋 예상되는 변경사항** 표시:

```
📋 예상되는 변경사항
- 처리할 회차 (3개): 39차 · 40차 · 41차
- 레이드 결과 행 추가: 39차, 40차, 41차
- 레이드 통계 신규 행: 총 284행
- 탈퇴 멤버 (0명)
- 신규 가입 멤버 (0명)
```

#### 2️⃣ 모든 변경 적용

변경사항을 검토한 후 **[✅ 모든 변경 적용]** 클릭

→ 다음이 처리할 회차별로 자동 진행:

1. 멤버 변동 처리 (탈퇴/신규 자동 sync)
2. `_backup_{생성시각}` 탭 자동 생성 + 원본 데이터 복사
3. 레이드 통계 탭에 빠진 회차 행을 차수 순서 위치에 추가
4. 유니온 멤버 탭에 빠진 회차 컬럼을 차수 순서 위치에 추가 + 그 회차 참가자의 싱크로 레벨 입력
5. 레이드 결과 탭에 빠진 회차 행 추가 (회차 컬럼만, 순위/머장 한줄평은 직접 입력)
6. (`탈퇴자 레벨 기록` 탭이 있으면) 유니온을 떠난 멤버의 회차별 레벨을 그 탭에 보존

완료 후:

```
✅ 39차·40차·41차 적용 완료 · backup 탭 _backup_20260601-143052
🎉 최신 회차 모든 멤버 참여 완료 — 미참여자 0명
```

최신 회차 미참여자가 있으면 `⚠️ 최신 회차 미참여 멤버 (N명): ...`로 표시되어 누가 빠졌는지 한눈에 확인 가능합니다.

### 멤버 변동이 있는 경우

탈퇴자 / 신규 가입자가 감지되면 변경사항 미리보기에 표시됩니다:

```
- 탈퇴 멤버 (1명): 옛멤버닉네임
- 신규 가입 멤버 (1명): 신규멤버닉네임
```

**[✅ 모든 변경 적용]** 클릭하면 자동으로:

- 탈퇴자 행의 아래 데이터를 한 칸씩 위로 이동 (Col A 가입 순서 1~32 유지)
- 신규 가입자를 빈 슬롯에 채움

## 5. ⚠️ 주의사항 (꼭 읽어주세요)

### ⚠️ 회차 관련

1. **빠진 과거 회차도 자동으로 채워집니다**
   - 도구는 시트에 없는 회차를 감지해 블라블라링크에서 가져와 보충합니다 (블라블라링크가 제공하는 과거 회차 범위 내).
   - 한 회차를 건너뛰어도 다음 사용 때 자동으로 메워지므로, 매 회차 꼭 사용하지 않아도 됩니다.

2. **회차 번호는 블라블라링크에서 자동 인식**
   - 블라블라링크의 회차 식별값(season_id)으로 정확한 회차 번호를 결정합니다. (시트 마지막 회차 +1 추측은 식별 실패 시의 예비 수단입니다.)

3. **이미 존재하는 회차의 데이터는 검증·수정하지 않습니다** (엣지 케이스)
   - 도구는 회차의 **존재 여부**(유니온 멤버 회차 컬럼 / 레이드 통계·결과 회차 라벨)만 보고 빠진 회차를 추가합니다.
   - 어떤 회차가 이미 있으면 — **셀이 비어있거나·값이 틀렸거나·일부만 채워져 있어도** — "이미 있음"으로 간주해 건드리지 않습니다 (덮어쓰기·보충 안 함).
   - **온전하지 않은 회차를 다시 채우려면**: 그 회차의 **유니온 멤버 컬럼 + 레이드 통계 행 + 레이드 결과 행을 모두 삭제**한 뒤 다시 실행하세요. 한 탭에서만 지우면 그 탭만 다시 채워집니다(탭별 판정).

### ⚠️ 멤버 변동 관련

4. **유니온 멤버는 최초 1회 사용자가 직접 입력**
   - 도구 최초 실행 시 시트의 닉네임을 기준으로 멤버를 매칭합니다.
   - **가입 순서대로 32명의 닉네임을 사용자가 직접 입력**해야 도구가 사용 가능합니다.
   - 1회 이후에는 B열에 숨겨진 member_id 열이 생성되어 member_id 기준으로 매칭합니다. (닉네임 변경되어도 추적 가능)
   - 닉네임 변경 시 유니온 멤버 탭에서는 갱신되지만, 레이드 통계는 따로 수정해야합니다.

5. **신규 가입자 여러 명일 때 가입 순서 정확하게 하려면 수동 조정**
   - 도구는 신규 가입자를 빈 슬롯에 위에서부터 자동으로 채웁니다.
   - 같은 회차에 여러 명이 가입한 경우, 그들 사이의 정확한 가입 순서를 반영하려면 사용자가 직접 수동 조정해야 합니다.
   - 다만 보통은 같은 회차 내 가입이라 큰 의미는 없습니다.

6. **탈퇴 판단은 "현재 유니온 소속" 기준** (레이드 참여 여부 아님)
   - 현재 유니온 멤버 목록(블라블라링크)에 없는 시트 멤버를 탈퇴자로 봅니다. 특정 회차에 레이드를 쉬었어도 유니온에 남아 있으면 탈퇴가 아닙니다.
   - **`탈퇴자 레벨 기록` 탭이 있는 시트**라면, 탈퇴자가 떠나기 전까지 쌓인 회차별 레벨을 그 탭(A열 닉네임 · B열+ 회차)에 보존합니다. 과거 백필에서 발견된 탈퇴자도 자동 기록됩니다. 탭이 없으면 이 기능은 건너뜁니다 (기존과 동일하게 동작).

### ⚠️ 시트 운영 관련

7. **레이드 결과 탭의 순위 / 머장 한줄평은 사용자 직접 입력**
   - 도구는 회차 컬럼만 자동 채웁니다. 순위와 머장 한줄평은 사용자가 직접 작성해야 합니다.

8. **백업 탭은 최근 3개만 유지**
   - 백업 탭은 매 실행마다 생성 시각 기반 이름(`_backup_YYYYMMDD-HHmmss`)으로 새로 만들어지며, 3개를 넘으면 가장 오래된 것이 자동 삭제됩니다.
   - 더 오래 보존하려면 직접 백업 탭 이름을 `_backup_` 외의 이름으로 변경해주세요.

9. **시트 헤더를 임의로 수정하지 마세요**
   - 헤더 컬럼명을 변경하면 시트 구조 검증에서 차단됩니다.
   - 신규 사본을 받으셨다면 신뢰 다이얼로그에서 체크리스트 검토 후 옵트인 가능합니다.

## 6. 문제 해결 (FAQ)

### Q. "사용자 스크립트 허용" 옵션을 못 찾겠어요

`chrome://extensions` → 우측 상단 **"개발자 모드"** 켜기 → Tampermonkey 카드에서 **"세부정보"** 클릭 → **"사용자 스크립트 허용"** 토글 ON.

### Q. 블라블라링크 새 탭이 열려도 도구로 데이터가 전송 안 됩니다

확인 사항:

1. Tampermonkey 확장이 활성화되어 있는지
2. Greasyfork 유저스크립트가 설치되어 있는지 (Tampermonkey 대시보드에서 확인)
3. 사용자 스크립트 허용 옵션이 켜져 있는지
4. 블라블라링크에 본인 NIKKE 계정으로 로그인되어 있는지

### Q. 시트 구조 미일치 알림이 떴는데 진행해도 되나요
- 이 알림이 발생했다면 이 도구와 시트가 호환되지 않는 상태일 가능성이 매우 높습니다..
- 본인이 정상 사본으로 신뢰한다면 신뢰 다이얼로그의 체크리스트를 검토 후 진행 가능합니다 ([Step 7](#step-7-신뢰-다이얼로그-처리-필요-시) 참고).
- 시트 헤더를 변경한 적 있다면 헤더를 원래대로 복원해주세요.
- **[변경]** 버튼으로 정상 사본 시트로 변경하길 추천합니다.
- 잘못된 시트에 도구 사용 시 시트 손상 위험있습니다. (다만 backup 탭으로 복원 가능).

### Q. 회차 번호가 틀리게 입력됐어요

- 회차 번호는 블라블라링크에서 자동 인식하므로 보통 어긋나지 않습니다.
- 그래도 수정이 필요하면 시트의 해당 컬럼/행을 직접 편집해주세요.
- 이미 입력된 회차의 **내용**이 잘못됐다면, 그 회차를 세 탭(유니온 멤버 컬럼·레이드 통계 행·레이드 결과 행)에서 모두 삭제하고 다시 실행하면 새로 채워집니다.

### Q. 잘못 입력됐어요. 되돌리고 싶어요

- `_backup_{생성시각}` 탭에서 원본 데이터를 확인할 수 있습니다.
- Google Sheets의 **파일 > 버전 기록 > 버전 기록 보기**에서 쓰기 이전 시점으로 복원 가능합니다.

### Q. 모바일에서 사용 가능한가요

- 미지원입니다.

### Q. Edge / Firefox / Safari 등 다른 브라우저에서도 사용 가능한가요

- **확인된 환경은 Chrome뿐입니다.**
- Edge는 Chromium 기반이라 기술적으로 작동 가능성이 높지만 검증되지 않았습니다.
- Firefox / Safari는 Tampermonkey 호환성과 일부 API 동작 차이로 동작이 보장되지 않습니다.

## 7. 개인정보 + 보안

### 데이터 처리 방식

- **외부 서버로 데이터 전송 0건** — 도구는 GitHub Pages 정적 호스팅으로 서버 측 처리가 없습니다.
- **Google Sheets API ↔ 사용자 본인 시트** — 사용자가 명시 선택한 시트만 접근. 다른 파일 / 드라이브 전체 접근 불가.
- **블라블라링크 데이터 ↔ 도구** — 같은 브라우저 내 cross-tab postMessage. 외부로 나가지 않음.

### OAuth 권한

- **drive.file scope** — 사용자가 명시 선택한 파일만 접근. Google에서 verification 면제하는 non-sensitive scope.
- **userinfo.email scope** — 로그인 이메일 표시용. 다른 개인정보 접근 불가.

### 토큰 보관

- Access token: **sessionStorage** (탭 종료 시 자동 소멸)
- 시트 ID, 매칭 캐시, 신뢰 hash: localStorage (민감 정보 X)
