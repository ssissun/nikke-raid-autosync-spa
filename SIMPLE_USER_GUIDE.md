# NIKKE 유레 자동 동기화 — 간편 사용 가이드

> 블라블라링크 유니온 레이드 회차 데이터를 본인 Google Sheets에 자동 입력합니다.
> **데스크탑 Chrome 전용**.

> 💡 **이미 NIKKE 유레 시트를 수동으로 운영 중이라면 Step 3, Step 4를 건너뛰셔도 됩니다.**
> 
> ✅ Step 1~4, 6은 **최초 1회만** 하면 됩니다.
---

## Step 1. Tampermonkey 설치 + 켜기

블라블라링크 데이터를 자동으로 가져오기 위한 확장 프로그램입니다.

1. Chrome에서 [Tampermonkey 설치](https://chromewebstore.google.com/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo)
2. 주소창에 `chrome://extensions` 입력 → 우측 상단 **"개발자 모드"** 켜기
3. **Tampermonkey** 카드 → **"세부정보"** → **"사용자 스크립트 허용"** 켜기

> 3번 설정을 안 하면 다음 Step의 스크립트가 작동하지 않습니다.

---

## Step 2. 자동 수집 스크립트 설치

[Greasyfork 페이지](https://greasyfork.org/scripts/579278) 접속 → **[설치하기]** → **[설치]** 클릭

---

## Step 3. NIKKE 유레 시트 본인 사본 만들기

> ⏭️ **이미 본인 사본을 운영 중이라면 건너뛰세요.**

1. [arca.live 게시글](https://arca.live/b/nikketgv/161405505)에서 NIKKE 유레 시트 원본 링크 확인
2. 원본 시트 접속 → `파일 > 사본 만들기`
3. 본인 Google Drive에 저장

---

## Step 4. 유니온 멤버 준비 (선택)

> ⏭️ **이미 32명 입력해서 운영 중이라면 건너뛰세요.**
>
> 🆕 **비워둔 채 시작해도 됩니다.** `유니온 멤버` 탭이 완전히 비어 있으면, 다음 단계에서 [신규 회차 데이터 가져오기]를 누를 때 도구가 닉네임을 자동으로 채워줍니다.

직접 입력하려면 본인 사본의 **`유니온 멤버`** 탭에 가입 순서대로 32명 닉네임을 넣습니다.

| A열 (가입 순서) | B열 (닉네임) |
|----|----|
| 1 | 1번째 멤버 닉네임 |
| 2 | 2번째 멤버 닉네임 |
| ... | ... |
| 32 | 32번째 멤버 닉네임 |

> ⚠️ **자동으로 채워진 닉네임은 가입 순서가 아닙니다.** 도구가 가입 순서를 알 수 없어 수집된 순서대로 넣습니다. 가입 순서가 중요하면 `유니온 멤버` 탭에서 직접 정렬해주세요.

---

## Step 5. 도구 페이지 접속 + 로그인

1. [도구 페이지 열기](https://ssissun.github.io/nikke-raid-autosync-spa/)
2. **[Google 로그인]** 클릭 → 본인 Google 계정 선택 → 권한 동의
3. 화면 상단에 `✅ 로그인됨 · [본인 이메일]` 표시 확인

---

## Step 6. 사본 시트 선택

1. **[시트 선택]** 클릭 → 시트 선택 창이 열림
2. 본인 사본 시트 선택 → **[선택]**
3. 자동으로 시트 검증 실행
4. `✅ 시트 구조 검증 통과` 표시 확인

> ⚠️ 다른 알림이 뜨면 [전체 가이드](./USER_GUIDE.md#step-7-신뢰-다이얼로그-처리-필요-시)의 Step 7 참고

---

## Step 7. 회차 데이터 가져오기 *(매 회차 반복)*

1. **[🎯 신규 회차 데이터 가져오기]** 클릭
2. 블라블라링크 새 탭이 열림 → 자동으로 데이터 수집되어 도구 페이지로 전달됨 (수집 중 도구 화면에 **진행 상태**가 표시됩니다 · 유저스크립트 v2.5.0+)
3. 도구 페이지에 **📋 예상되는 변경사항**이 표시됨 (시트에 빠진 과거 회차가 여러 개면 한 번에 함께 가져옴):

   ```
   - 처리할 회차 (3개): 39차 · 40차 · 41차
   - 레이드 통계 신규 행: 총 284행
   - 탈퇴 멤버 (0명)
   - 신규 가입 멤버 (0명)
   ```

   > 탈퇴·신규·닉네임 변경이 있으면 각 멤버에 **시트 행 번호와 +/−/~ 표시**가 함께 나옵니다 (예: `− 탈퇴자1 (행 32)`, `+ 새멤버`, `~ 옛닉 (행 5) → 새닉`). 같은 닉네임이 여러 명이어도 행 번호로 구분됩니다.

---

## Step 8. 모든 변경 적용 *(매 회차 반복)*

변경사항을 확인한 후 **[✅ 모든 변경 적용]** 클릭

→ 처리할 회차별로 다음이 자동 진행됩니다:

- 유니온 멤버 탭에 회차 컬럼 추가 + 그 회차 참가자의 싱크로 레벨 입력
- 레이드 통계 탭에 회차 행 추가
- 레이드 결과 탭에 회차 행 추가
- 탈퇴/신규 멤버 자동 처리
- (`탈퇴자 레벨 기록` 탭이 있으면) 탈퇴자의 회차별 레벨 보존
- `_backup_{생성시각}` 탭 자동 생성 (원본 데이터 백업)

완료 후 화면에 최신 회차 미참여 멤버가 표시됩니다 (없으면 "미참여자 0명").

---

## ⚠️ 꼭 알아둘 것

- **빠진 과거 회차는 자동으로 채워집니다.** 회차 번호도 블라블라링크에서 자동 인식하므로, 한 회차 건너뛰어도 다음 사용 때 자동으로 메워집니다.
- **이미 있는 회차의 데이터는 고치지 않습니다.** 회차 컬럼/행이 이미 있으면 셀이 비었거나 틀렸어도 그대로 둡니다. 다시 채우려면 그 회차를 **세 탭(유니온 멤버·레이드 통계·레이드 결과)에서 모두 삭제** 후 다시 실행하세요.
- **레이드 결과 탭의 순위 / 머장 한줄평은 사용자가 직접 입력해야 합니다.** 도구는 회차 컬럼만 채웁니다.
- **탈퇴자 레벨 기록은 `탈퇴자 레벨 기록` 탭이 있는 시트에서만** 동작합니다. 탈퇴 판단은 현재 유니온 소속 여부 기준입니다.
- **백업 탭은 최근 3개만 유지**됩니다 (생성 시각 기반 이름 `_backup_YYYYMMDD-HHmmss`). 더 보존하려면 탭 이름을 `_backup_` 외의 이름으로 변경하세요.

## 🔧 안 될 때

| 증상 | 해결 |
|------|------|
| 블라블라링크 새 탭이 열려도 도구로 데이터 전송 안 됨 | Step 1의 **"사용자 스크립트 허용"** 옵션 확인. Tampermonkey + 유저스크립트 모두 활성화 |
| 시트 구조 미일치 알림 | [전체 가이드](./USER_GUIDE.md#step-7-신뢰-다이얼로그-처리-필요-시) Step 7 참고 |
| 잘못 입력됨 | `_backup_{생성시각}` 탭에서 원본 확인. Google Sheets `파일 > 버전 기록`에서 복원 |
| 모바일에서 안 됨 | 미지원 — 데스크탑 Chrome 사용 |
| Edge / Firefox / Safari | Chrome만 확인됨. 다른 브라우저는 보장 안 됨 |
